Mão na Massa Terraform

Por: Vinicius Zenha

Objetivo:

Planejar, criar e lançar a infra para um serviço.

Roteiro:

  • Entender o problema
  • Pensar em uma solução
  • Planejar a implementação
  • Implementar (Terraform útil a partir daqui)
  • Lançar

O Problema

Criar uma infraestrutura serverless para um Guestbook Online (Mural de Mensagens).

Usuários podem visualizar e postar mensagens simples.

Foco: Infraestrutura provisionada via Terraform.

Use Case Diagram

Design de Sistema – Visão Genérica

Arquitetura da Solução na AWS

  • CloudFront → CDN
  • S3 → Static File Storage
  • API Gateway → API Gateway
  • Lambda → FaaS
  • DynamoDB → NoSQL Database

Overview Terraform

  • Infrastructure as Code (IaC)
  • Provisionamento declarativo
  • Providers (AWS, Azure, GCP...)
  • Resources, Modules, Variables, Outputs
  • State Management (.tfstate)
  • Principais comandos: init, plan, apply

Componentes Terraform por Serviço

Serviço Recurso Terraform
S3 aws_s3_bucket
CloudFront aws_cloudfront_distribution
API Gateway aws_api_gateway_rest_api
Lambda aws_lambda_function
DynamoDB aws_dynamodb_table
IAM aws_iam_role, aws_iam_policy

Fluxo de Deploy com Terraform

  1. terraform init
  2. terraform plan
  3. terraform apply

Resultado esperado: Infraestrutura provisionada e API pública.
